# 首页工具列表与搜索优化设计

## 背景

当前公开首页主要展示 Hero、分类入口和少量精选工具。用户需要进入某个分类页后，才能看到该分类下的完整工具列表。工具卡片会展示“含联盟链接”，分类页也提供“有 Affiliate”筛选，这些属于内部运营信息，不应出现在公开网站中。

搜索页目前支持按名称、一句话介绍和详细介绍做包含匹配，但没有覆盖标签，也不支持多个关键词组合搜索。

## 目标

1. 首页直接展现各个分类下的全部工具，让用户不需要进入分类页也能浏览工具。
2. 公开页面不展示任何联盟相关标签或筛选项。
3. 搜索支持更宽的工具内容匹配，包括名称、标签、一句话介绍和详细介绍。
4. 搜索支持多个关键词组合匹配，所有关键词都需要在工具内容集合中命中。

## 非目标

1. 不引入拼写纠错、拼音、首字母搜索或外部搜索服务。
2. 不调整后台联盟管理、跳转链接选择或统计埋点。
3. 不重做整站视觉风格，只优化首页信息架构和现有组件展示。

## 首页信息架构

首页保留 Hero 区域，但减少“点击分类进入”的依赖。Hero 下方增加或改造为分类快速导航，用于跳转到页面内对应分类锚点。

分类内容区按 `categories.priority` 顺序渲染。每个分类作为一个独立专区，包含分类 emoji、名称、描述、工具数量，以及该分类下全部已发布工具。每个专区的工具使用现有 `ToolCard` 样式延展成网格布局。

分类页继续保留，作为独立 URL 和 SEO 页面存在。首页工具卡片和分类页工具卡片应共享同一套公开展示规则。

## 工具卡片展示

`ToolCard` 不再在公开页面展示“含联盟链接”徽标。`showAffiliateBadge` 这类公开展示开关可以删除或固定为不展示，避免后续调用方误用。

工具访问链接仍按现有逻辑优先使用 `urlAffiliate`，没有联盟链接时使用 `urlOfficial`。`data-has-affiliate` 等统计属性可保留，因为它们服务于内部统计，不作为可见 UI 文案展示。

分类页移除“有 Affiliate”筛选 Tab，只保留公开用户可理解的筛选项：全部、免费、付费/Freemium。

## 搜索行为

搜索输入先 `trim`，再按连续空白拆成关键词列表。查询为空时不展示结果。

每个工具构建一个可搜索内容集合：

- `name`
- `oneLiner`
- `description`
- `tags`

匹配规则为：每个关键词都需要在上述任一字段中命中。英文大小写不敏感；中文和其他字符按包含匹配处理。

搜索页从数据库读取全部已发布工具后，在应用层用统一 matcher 过滤。这样可以同时覆盖 `tags` JSON 字段，并确保 SQLite、Postgres 和 fallback JSON 数据的搜索行为一致。当前工具规模适合这种实现；后续工具数量明显增长时，再考虑数据库全文索引或专门搜索服务。

## 数据获取

首页数据查询从“分类 + 精选工具 + 分类计数”调整为“分类 + 全部已发布工具”。工具按 `sortOrder` 排序，并在页面层按 `categoryId` 分组。

fallback 数据沿用 `data/categories.json` 和 `data/tools.json`，映射字段与现有 `public-data.ts` 保持一致。数据库超时后仍回退到 JSON 数据。

## 错误与空状态

如果某个分类下没有工具，首页可不展示该分类的工具网格，或展示轻量空状态。推荐不展示空分类，保持首页紧凑。

搜索无结果时沿用当前空状态，提示没有找到相关工具，并保留提交新工具入口。

数据库查询失败或超时时，继续使用 fallback 数据，不改变用户可见错误体验。

## 验收标准

1. 打开首页即可看到每个分类下的全部已发布工具。
2. 首页、分类页、搜索页的工具卡片不显示“含联盟链接”。
3. 分类页不再显示“有 Affiliate”筛选。
4. 搜索 `Perplexity`、`AI 搜索`、`市场调研` 等名称、标签或介绍内容都能找到相关工具。
5. 搜索多个关键词时，例如 `etsy 关键词`，只返回内容集合中同时匹配两个关键词的工具。
6. 工具详情页和外链跳转行为保持不变。

## 实现影响范围

预计涉及以下文件：

- `app/(public)/page.tsx`
- `app/(public)/category/[slug]/page.tsx`
- `app/(public)/search/page.tsx`
- `components/tool-card.tsx`
- 可选新增首页分类工具列表组件，例如 `components/category-tool-sections.tsx`
- 可选提取搜索 matcher 到 `lib/public-data.ts`
